# Развертывание Telegram бота на Timeweb Cloud

## Подготовка к развертыванию

### 1. Настройка переменных окружения

Скопируйте файл `timeweb.env` и настройте переменные:

```bash
cp timeweb.env .env.timeweb
```

Обязательно измените следующие переменные:

- `BOT_TOKEN` - токен вашего бота от @BotFather
- `CHANNEL_ID` - ID канала для уведомлений
- `WEBHOOK_BASE_URL` - URL вашего приложения на Timeweb Cloud (замените `your-app-name` на реальное имя)
- `WEBHOOK_SECRET` - секретный ключ для webhook
- `DB_PASS` - пароль для PostgreSQL
- `REDIS_PASS` - пароль для Redis
- `DEFAULT_ADMIN_PASSWORD` - пароль для админ-панели
- `SECRET_KEY` - секретный ключ для Flask

### 2. Структура файлов для Timeweb Cloud

Используйте следующие файлы:
- `docker-compose.timeweb.yml` - основной файл для развертывания
- `Dockerfile` - образ бота (уже настроен)
- `.env.timeweb` - переменные окружения

## Развертывание на Timeweb Cloud

### Шаг 1: Подготовка репозитория

1. Убедитесь, что все файлы загружены в ваш Git репозиторий
2. Проверьте, что файл `docker-compose.timeweb.yml` находится в корне репозитория

### Шаг 2: Создание приложения в Timeweb Cloud

1. Войдите в панель управления Timeweb Cloud
2. Перейдите в раздел **Apps**
3. Нажмите **"Создать приложение"**
4. Выберите тип **"Docker Compose"**

### Шаг 3: Подключение репозитория

1. Подключите ваш Git репозиторий (GitHub, GitLab, BitBucket)
2. Выберите ветку для деплоя (обычно `main` или `master`)
3. Укажите путь к файлу Docker Compose: `docker-compose.timeweb.yml`

### Шаг 4: Настройка переменных окружения

В разделе **"Переменные окружения"** добавьте все переменные из файла `.env.timeweb`:

```
BOT_TOKEN=your_bot_token_here
CHANNEL_ID=your_channel_id_here
USE_WEBHOOK=true
WEBHOOK_BASE_URL=https://your-app-name.timeweb.cloud
WEBHOOK_SECRET=your_webhook_secret_here
WEBHOOK_HOST=0.0.0.0
WEBHOOK_PORT=8080
DB_HOST=postgres
DB_PORT=5432
DB_USER=postgres
DB_PASS=your_secure_password_here
DB_NAME=tgbot
REDIS_HOST=redis
REDIS_PORT=6379
REDIS_PASS=your_redis_password_here
ADMIN_HOST=0.0.0.0
ADMIN_PORT=5000
DEFAULT_ADMIN_EMAIL=admin@yourdomain.com
DEFAULT_ADMIN_PASSWORD=your_admin_password_here
SECRET_KEY=your_secret_key_here
DEBUG=false
INTEGRATIONS_STUB=true
RATE_LIMIT=0.5
EMAIL_TOKEN_TTL_MINUTES=15
FORTUNE_APP_URL=https://fortunewheelsinglefile.netlify.app/
NOTCOIN_APP_URL=https://not-coin-mini-app.vercel.app/
SENTRY_DSN=
AMPLITUDE_API_KEY=
```

### Шаг 5: Настройка сервера

1. Выберите регион размещения
2. Выберите конфигурацию сервера (рекомендуется минимум 1 vCPU, 1GB RAM)
3. Настройте домен (если нужно)

### Шаг 6: Запуск деплоя

1. Нажмите **"Запустить деплой"**
2. Дождитесь завершения сборки и запуска контейнеров
3. Проверьте логи в разделе **"Деплой"**

## Проверка работы

### 1. Проверка бота

После успешного деплоя:
1. Откройте URL вашего приложения: `https://your-app-name.timeweb.cloud`
2. Проверьте, что бот отвечает на команды
3. Проверьте webhook: `https://your-app-name.timeweb.cloud/webhook`

### 2. Проверка админ-панели

Админ-панель доступна по адресу: `https://your-app-name.timeweb.cloud:8081`

### 3. Проверка базы данных

База данных PostgreSQL доступна внутри контейнеров по адресу `postgres:5432`

## Особенности Timeweb Cloud

### Ограничения портов

- Порты 80 и 443 зарезервированы под Nginx
- Первый сервис в docker-compose.yml автоматически получает проксирование
- Остальные сервисы доступны с явным указанием порта

### Вольюмы

- Вольюмы не сохраняются при повторном деплое
- Для постоянного хранения данных используйте внешние сервисы

### Мониторинг

- Встроенный мониторинг Timeweb Cloud
- Логи доступны в панели управления
- Метрики производительности в разделе "Мониторинг"

## Устранение неполадок

### Проблемы с webhook

1. Проверьте, что `WEBHOOK_BASE_URL` указан правильно
2. Убедитесь, что `USE_WEBHOOK=true`
3. Проверьте логи бота на наличие ошибок

### Проблемы с базой данных

1. Проверьте, что миграции выполнились успешно
2. Убедитесь, что контейнер `migrator` завершился без ошибок
3. Проверьте подключение к базе данных

### Проблемы с Redis

1. Проверьте, что Redis запустился успешно
2. Убедитесь, что пароль указан правильно
3. Проверьте логи Redis контейнера

## Обновление приложения

1. Внесите изменения в код
2. Зафиксируйте изменения в Git
3. В панели Timeweb Cloud нажмите **"Пересобрать"**
4. Дождитесь завершения деплоя

## Резервное копирование

Рекомендуется настроить автоматическое резервное копирование базы данных:

1. Используйте внешний сервис для хранения бэкапов
2. Настройте периодическое создание дампов
3. Храните бэкапы в облачном хранилище

## Безопасность

1. Используйте сложные пароли для всех сервисов
2. Регулярно обновляйте зависимости
3. Мониторьте логи на предмет подозрительной активности
4. Используйте HTTPS для всех внешних подключений
