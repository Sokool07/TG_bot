# Развертывание бота на Timeweb Cloud

## ✅ Статус: Готово к развертыванию

Бот полностью подготовлен для развертывания на Timeweb Cloud с внутренними БД и без внешних подключений (кроме miniapp).

## Подготовка к развертыванию

### 1. Создание приложения в Timeweb Cloud

1. Перейдите в панель управления Timeweb Cloud
2. Создайте новое приложение в разделе "Apps"
3. Выберите "Деплой из Dockerfile"
4. Подключите репозиторий с проектом

### 2. Настройка базы данных

1. Создайте кластер PostgreSQL в Timeweb Cloud
2. Выберите "Приватный доступ" для безопасности
3. Получите параметры подключения:
   - Хост базы данных
   - Порт (обычно 5432)
   - Имя пользователя
   - Пароль
   - Имя базы данных

### 3. Настройка Redis

1. Создайте кластер Redis в Timeweb Cloud
2. Выберите "Приватный доступ"
3. Получите параметры подключения:
   - Хост Redis
   - Порт (обычно 6379)
   - Пароль (если требуется)

### 4. Настройка переменных окружения

Скопируйте `timeweb.env.example` в `.env` и заполните переменные:

```bash
cp timeweb.env.example .env
```

Обязательные переменные:
- `BOT_TOKEN` - токен вашего Telegram бота
- `CHANNEL_ID` - ID канала для бота
- `WEBHOOK_BASE_URL` - URL вашего приложения в Timeweb Cloud
- `WEBHOOK_SECRET` - секретный ключ для webhook
- `DB_HOST`, `DB_USER`, `DB_PASS`, `DB_NAME` - параметры PostgreSQL
- `REDIS_HOST`, `REDIS_PASS` - параметры Redis

### 5. Развертывание

1. Загрузите код в репозиторий
2. Timeweb Cloud автоматически соберет Docker образ
3. Приложение будет доступно по URL: `https://your-app-name.timeweb.cloud`

## Особенности конфигурации

### Отключенные внешние зависимости

Для работы без внешних подключений отключены:
- Amplitude Analytics
- Google Analytics  
- PostHog Analytics
- Sentry (опционально)
- Внешние API (CustomerIO, GBL Bonus, WS SV)

### Сохраненные внешние подключения

Оставлены только miniapp:
- Fortune App (https://fortunewheelsinglefile.netlify.app/)
- Notcoin App (https://not-coin-mini-app.vercel.app/)

### Проверка готовности

Docker образ успешно собран и протестирован:
```bash
docker build -t tgbot-timeweb .
```

### Webhook настройки

- Порт: 8080 (стандартный для Timeweb Cloud)
- Хост: 0.0.0.0 (слушает все интерфейсы)
- Путь: /webhook

## Мониторинг

После развертывания проверьте:
1. Логи приложения в панели Timeweb Cloud
2. Подключение к базе данных
3. Подключение к Redis
4. Работу webhook через Telegram API

## Безопасность

- Все внутренние сервисы используют приватный доступ
- Внешние подключения ограничены только miniapp
- Webhook защищен секретным ключом
